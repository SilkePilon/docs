### Real-time Updates WebSocket
Connect to our community-maintained WebSocket endpoint to receive instant updates and participate in article discussions.

**WebSocket URL:**
<code>wss://api.jotihunt.live/ws</code>

**Features:**
- Real-time game updates (hints, assignments, articles)
- Article discussion chat system
- No authentication required
- Automatic reconnection
- Low latency delivery
- Unlimited connections
</Check>

<Note>
### Game Updates Channel
Here's an example of connecting to the game updates channel:

```javascript
const ws = new WebSocket('http://ws.jotihunt.live');

ws.onopen = () => {
  console.log('Connected to Jotihunt live updates');
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  
  switch (data.type) {
    case 'hint':
      console.log('New hint for area:', data.data.area);
      console.log('Hint content:', data.data.content);
      break;
      
    case 'assignment':
      console.log('New assignment:', data.data.title);
      console.log('Points:', data.data.points);
      console.log('Description:', data.data.description);
      break;
      
    case 'article':
      console.log('New article:', data.data.title);
      console.log('Category:', data.data.category);
      break;
  }
};

// Error handling
ws.onerror = (error) => {
  console.error('WebSocket error:', error);
};

ws.onclose = () => {
  console.log('Connection closed, attempting to reconnect...');
  setTimeout(() => {
    // Reconnect logic
  }, 5000);
};
```
</Note>


<Warning>
### Implementation Guidelines
- Implement proper error handling and reconnection logic
- Handle all message types appropriately
- Keep track of connection state
- Consider implementing an exponential backoff for reconnection attempts
</Warning>
